Console phrasings

Type any of these on /console. Marco routes to the matching tool.

  • show me the repo status → workspace_status
  • find all Python files → find_files
  • where do we use the database? → lookup_content
  • remember: we use JWT for service auth → save_memory
  • plan a refactor of the auth flow → create_plan
  • stage a patch that adds a version line to README.md → suggest_patch

Safety rules

  • Applying a patch requires typing the patch name in the confirm box.
  • Running a script requires {execute:true, confirm:true} and an allow-listed prefix.
  • Rollbacks are one-click but also logged to ~/.marco/audit.log.
  • The HTTP server forces safety_mode=workspace-write. danger-full-access is not reachable over HTTP.
  • The console never applies patches or runs scripts — those still require the Patches / Scripts UI.

CLI equivalents

From a shell on the Droplet — all routed through src/marco_v3/cli.py.

  • $ python3 -m src.main doctor
  • $ python3 -m src.main status
  • $ python3 -m src.main inspect <query>
  • $ python3 -m src.main plan <goal>
  • $ python3 -m src.main execute <session>
  • $ python3 -m src.main validate <session>
  • $ python3 -m src.main recover <session>
  • $ python3 -m src.main patch propose|apply|rollback
  • $ python3 -m src.main memory add|list|recall
  • $ python3 -m src.main scaffold page|component|route|service

Nautical glossary

Workspace — a registered repo. Marco steers one at a time.
Session — a multi-phase task (plan → execute → validate → recover).
Patch — a typed find/replace with an automatic checkpoint.
Memory — workspace-scoped notes / decisions / conventions.
Script — an allow-listed shell command, dry-run by default.
Audit — logbook of every mutation Marco made.
